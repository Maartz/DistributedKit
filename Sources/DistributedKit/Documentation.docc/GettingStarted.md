# Getting Started with DistributedKit

Build your first OTP-style distributed application in Swift.

## Overview

This guide walks through building a complete DistributedKit application: defining actors with `@Service`, composing them into a supervision tree, and running the application with graceful shutdown support.

### Prerequisites

- Swift 6.3+
- macOS 15+ or Linux
- Add DistributedKit to your `Package.swift`:

```swift
dependencies: [
    .package(url: "https://github.com/alembic-labs/DistributedKit.git", branch: "master"),
]
```

And add the target dependency:

```swift
.target(
    name: "MyApp",
    dependencies: [
        .product(name: "DistributedKit", package: "DistributedKit"),
    ]
)
```

## Step 1: Define a Service Actor

Use the `@Service` macro to define a distributed actor with supervision metadata:

```swift
import DistributedKit
import DistributedCluster

@Service(name: "counter", restart: .permanent)
distributed actor Counter {
    typealias ActorSystem = ClusterSystem

    private var count: Int = 0

    distributed func increment() -> Int {
        count += 1
        return count
    }

    distributed func get() -> Int {
        count
    }
}
```

The `@Service` macro generates:

- A `childSpec()` static method for supervision tree registration
- A `DistributedKitService` extension with `serviceName` and `restartStrategy`

### What `@Service` Expands To

```swift
// Generated by @Service(name: "counter", restart: .permanent)
extension Counter {
    static func childSpec() -> ChildSpec<Counter> {
        ChildSpec(
            name: "counter",
            restart: .permanent,
            factory: { system in try Counter(actorSystem: system) }
        )
    }
}

extension Counter: DistributedKitService {
    static var serviceName: String { "counter" }
    static var restartStrategy: RestartStrategy { .permanent }
}
```

## Step 2: Build a Supervision Tree

Compose actors into a supervision tree using the result builder DSL:

```swift
let tree = SupervisorTree("MyApp") {
    Supervisor("workers", strategy: .oneForOne) {
        Counter.childSpec()
        Logger.childSpec()
    }

    Supervisor("infrastructure", strategy: .oneForAll) {
        DatabasePool.childSpec()
        CacheManager.childSpec()
    }
}
```

### Supervision Strategies

| Strategy | Behavior |
|----------|----------|
| `.oneForOne` | Only the crashed child is restarted |
| `.oneForAll` | All children are restarted when one crashes |
| `.restForOne` | The crashed child and all children started after it are restarted |

### Rate Limiting

Each supervisor has a `maxRestarts` / `withinSeconds` rate limiter. If a child crashes too many times within the window, the supervisor shuts down:

```swift
Supervisor(
    "workers",
    strategy: .oneForOne,
    maxRestarts: 5,
    withinSeconds: 10
) {
    Counter.childSpec()
}
```

## Step 3: Run the Application

### High-Level: DistributedKitApplication

The simplest entry point — boots a `ClusterSystem`, starts the tree, handles SIGTERM/SIGINT:

```swift
// main.swift
import DistributedKit

try await DistributedKitApplication(
    name: "MyApp",
    clusterSettings: { settings in
        settings.bindPort = 7000
    }
) {
    Supervisor(strategy: .oneForOne) {
        Counter.childSpec()
    }
}.run()
```

### Low-Level: ServiceGroup Composition

For full control, compose `ClusterSystemService` and `BoundSupervisorTree` with
a `ServiceGroup` from `swift-service-lifecycle`:

```swift
import DistributedKit
import ServiceLifecycle
import Logging

let system = await ClusterSystem("MyApp") { settings in
    settings.bindPort = 7000
}

let tree = SupervisorTree("MyApp") {
    Supervisor(strategy: .oneForOne) {
        Counter.childSpec()
    }
}

let group = ServiceGroup(
    services: [
        ClusterSystemService(system),
        tree.bind(to: system),
    ],
    gracefulShutdownSignals: [.sigterm, .sigint],
    logger: Logger(label: "my-app")
)
try await group.run()
```

## Step 4: Discover Services

### Registry

Use `Registry` for typed service discovery:

```swift
let registry = Registry(system: system)
let key = ServiceKey<Counter>(id: "main-counter")

// Register
await registry.register(counter, key: key)

// Lookup
if let counter = await registry.lookup(key) {
    let value = try await counter.get()
}
```

### Singleton

Use `Singleton` for services registered via `@Service`:

```swift
let counter = try await Singleton<Counter>.resolve(on: system)
let value = try await counter.increment()
```

## Step 5: Test with TestKit

DistributedKit provides `DistributedKitTestKit` for testing:

```swift
import DistributedKitTestKit
import Testing

@Test func counterIncrements() async throws {
    try await withCluster("test") { system in
        let counter = Counter(actorSystem: system)
        let value = try await counter.increment()
        #expect(value == 1)
    }
}
```

### TestProbe

`TestProbe` is a distributed actor that captures messages for assertions:

```swift
let probe = TestProbe<Int>(actorSystem: system)
try await probe.send(42)
let received = try await probe.expectMessage()
#expect(received == 42)

// Assert no messages arrive within a timeout
try await probe.expectNoMessage(for: .seconds(1))
```
